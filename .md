## Convention cốt lõi

### Cache busting
- Khi sửa JS/CSS: BẮT BUỘC bump `?v=2.x` trong `index.html`
- Production dùng Cloudflare Pages → cache aggressive → quên bump = bug ẩn cho user

### Commit
- KHÔNG dùng `git add -A` hoặc `git add .` — chỉ add file cụ thể (tránh kéo theo rác)
- Commit message bằng tiếng Việt OK, có Co-Authored-By footer
- Tạo backup tag trước thao tác risky (`git tag backup-pre-X`)
- KHÔNG tự push — luôn hỏi user trước

### `.gitignore`
- `.sql` files → KHÔNG push GitHub (Supabase schema riêng tư)
- `Bao_cao_*.md`, `Bug_Report_*.md`, `Optimization_Plan_*.md` → local only
- `implementation_plan.md`, `DEPLOY_GUIDE.md`, `hsk_game_spec.md` → local only
- `.claude_bootstrap.md` (file này) → local only

### Workflow code → review → push

```
1. Plan + viết prompt chi tiết
2. Opus 4.6 implement (session riêng) hoặc tự code
3. Sonnet 4.6 verify (session này, hoặc cùng session)
4. Test local nhanh — chỉ test 4-5 critical paths, KHÔNG test exhaustive (tốn token)
5. Commit sạch (chỉ files liên quan)
6. User push thủ công
```

### Multi-model strategy
- **Opus 4.6**: implement task lớn (>3 files), logic phức tạp
- **Sonnet 4.6**: verify, bug fix nhỏ, prompt writing, cleanup
- **Opus 4.7**: chỉ khi Sonnet không tự tin (deep debug, kiến trúc)

## Style trả lời
- Khi user hỏi mơ hồ → đưa **2-4 option A/B/C/D** với tradeoff rõ, KHÔNG tự quyết
- Khi present plan/decision → bullet/table ngắn, không paragraphs dài
- Khi báo cáo task xong → 1 bảng kết quả + next step gợi ý
- KHÔNG tóm tắt lại những gì user vừa nói
- KHÔNG xác nhận lặp lại mỗi câu

## Anti-patterns (đừng làm)

- `git add -A` rồi kéo theo `.claude/worktrees/*`, file rác
- Push thẳng main mà chưa test
- Commit MD/SQL files lên GitHub
- Đọc `js/supabase.js`/`js/auth.js`
- Tự "cải tiến" thêm feature ngoài scope
- Lặp lại verbose context user đã biết
- Test exhaustive khi chỉ cần smoke test
- "Should I proceed?" — cứ làm nếu rõ ràng
- Anti-DevTools / F12 protection — security theater

## Patterns ĐÚNG

- Backup tag trước reset/filter-branch
- Verify code thực tế (grep + read) trước khi tin báo cáo Opus
- Bump cache version đồng bộ tất cả file đã sửa
- Khi gặp lỗi sync/data: check schema migration đã run chưa
- Khi gặp UI lỗi prod nhưng OK local: nghi cache CDN
- Tách worktree riêng cho task song song (nếu cần)
- Memory: ghi feedback user khi họ correct mình; ghi project decisions
